## Задание 1

**Создайте свой шаблон, в котором будут элементы данных, мониторящие загрузку CPU и RAM хоста.**

### Выполненные этапы:

1. **Создание шаблона:**
   - В разделе **Data collection > Templates** создан новый шаблон
   - Название шаблона: `My Linux CPU and RAM Template`
   - Шаблон добавлен в группу `Templates`

2. **Создание Item для мониторинга загрузки CPU:**
   - Название: `CPU utilization`
   - Тип: `Zabbix agent`
   - Ключ: `system.cpu.util[,,avg1]`
   - Тип информации: `Numeric (float)`
   - Единица измерения: `%`
   - Интервал обновления: `1m`

3. **Создание Item для мониторинга загрузки RAM:**
   - Название: `Memory utilization`
   - Тип: `Calculated` (вычисляемый элемент)
   - Ключ: `memory.usage.percent`
   - Формула: `100 - last(/{HOST.HOST}/vm.memory.size[pavailable])`
   - Тип информации: `Numeric (float)`
   - Единица измерения: `%`
   - Интервал обновления: `1m`

4. **Добавление тегов для удобной фильтрации:**
   - К обоим Items добавлен тег: `my-template: cpu-ram`
   - Теги позволяют быстро фильтровать метрики из нашего шаблона

## Задание 2-3

**Добавьте в Zabbix два хоста и привяжите к ним созданный шаблон и шаблон Linux by Zabbix Agent.**

### Выполненные этапы:

1. **Настройка существующих хостов:**
   - Использованы уже настроенные хосты из предыдущей работы:
     - `Zabbix-serv-Agent` (локальная VM)
     - `Docker Container Agent` (Docker контейнер)

2. **Привязка шаблонов к хостам:**
   - К обоим хостам привязан стандартный шаблон **Linux by Zabbix agent**
   - К обоим хостам привязан созданный шаблон **My Linux CPU and RAM Template**
   - Шаблоны привязаны через раздел **Templates** в настройках каждого хоста

3. **Проверка сбора данных:**
   - В разделе **Monitoring > Latest data** отображаются данные с обоих Items
   - Данные поступают с обоих хостов (локального и Docker)
   - Значения обновляются каждую минуту согласно заданным интервалам
   - Оба хоста имеют зелёный статус подключения

## Задание 4

**Создайте свой кастомный дашборд.**

### Выполненные этапы:

1. **Создание дашборда:**
   - В разделе **Dashboards > All dashboards** создан новый дашборд
   - Название дашборда: `My Hosts Resources`
   - Владелец: `Admin`

2. **Добавление виджетов (графиков):**
   - **Widget 1:** `CPU Utilization` - график загрузки CPU для хоста `Zabbix-serv-Agent`
   - **Widget 2:** `Memory utilization` - график загрузки RAM для хоста `Zabbix-serv-Agent`
   - **Widget 3:** `CPU and Memory for Docker Container Agent` - комбинированный график загрузки CPU и RAM для Docker хоста

3. **Настройка графиков:**
   - Использованы виджеты типа **Graph**
   - Для каждого графика настроены соответствующие Items
   - Для комбинированных графиков использованы **Overrides** для настройки цветов линий

4. **Особенности реализации:**
   - На дашборде отображаются метрики с обоих хостов
   - Графики обновляются в реальном времени с интервалом 1 минута
